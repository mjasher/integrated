calcLowFlowIndex(yearly_flow_data, flow_col, summer_low_thd, winter_low_thd, summer_index_thd, winter_index_thd)[source]¶

calculate low flow index:

  1. Identify climate condition of the input scenario (dry, average or wet) (TO DO, FOR TOY MODEL JUST USE AS AN INPUT)

  2. Extract summer and winter low flow thresholds based on climate condition, as per table below:

    Condition Summer_low Winter_low
    Month Dec-May Jun-Nov
    Dry climate >= 10 ML/day >=50 ML/day
    Avg climate >= 20 ML/day >= 100 ML/day
    Wet climate >= 50 ML/day >= 200 ML/day
    Minimum days 150 days 150 days

  3. For each year calculate the number of days between 1st Dec and 31st May that flow is >= summer low flow threshold, and the number of days between 1st Jun and 30st Nov that flow is >= winter low flow threshold.

  4. For each year, if the minimum days for summer AND winter low flows are met, then low flow index for platypus = 1, else, index = 0.

Parameters:
  • yearly_flow_data – Pandas dataframe of daily flow data for the given year in ML/Day
  • flow_col – Name of flow data column
  • summer_low_thd – flow threshold (in ML/day) above which summer low flow requirememnt is satisfied
  • winter_low_thd – flow threshold (in ML/day) above which winter low flow requirement is satisfied
  • summer_index_thd – a threshold (in days) above which the number of days summer low are satisfied
  • winter_index_thd – a threshold (in days) above which the number of days summer low are satisfied
Returns:

lowflow index with a value of 0 or 1.
calcBurrowIndex(yearly_flow_data, flow_col, burrow_startmonth, burrow_endmonth, entrance_buffer, breeding_startmonth, breeding_endmonth)[source]¶

Given specs: 1. Within each burrowing season window (1 July-31 Aug), identify max flow level (ML/day). 2. Calculate the platypus flow benchmark for this season: benchmark = max flow (ML/day) + burrow buffer (x ML/day). x ranges from 0 to a certain positive number to account for platypus barrow a little above the max flow. 3. Following each burrowing season, scan the breeding season (1st Sep-28th Feb), events are identified if flow reaches above the benchmark. 4. The longest event is used to calculate burrow flood index. If the duration of the duration is below or equal 1 days, the index is 1 (indicating the event is fine for platypus); if the duration is between 2-3 days, the index is 0.5, indicating a lower preference due to risk of drowning; if duration is 4 days or above, the index value is zero indicating the worst outcome for platypus due to flooding of the burrows.

Parameters:
  • yearly_flow_data – Pandas dataframe of daily flow data for the given year in ML/Day
  • flow_col – Name of flow data column
  • burrow_startmonth – the starting month of platypus burrowing period
  • burrow_endmonth – the end month of platypus burrowing period,
  • entrance_buffer – some buffer (ML/day) to add to max flow, allowing burrow entrance to be slightly higher than the max flow level
  • breeding_startmonth – the starting month of platypus breeding period
  • breeding_endmonth – the ending month of platypus breeding period
Returns:

burrow index with a value of 0 or 1

calcWaterSuitabilityIndex(gw_data, sw_data, gw_weight, sw_weight)[source]¶

\(I = w_{g}G + w_{s}S\)

where

  • \(I\) denotes the water suitability index
  • \(G\) is the groundwater suitability index
  • \(S\) is the surface water suitability index
  • \(w_{g}\) weight for groundwater index
  • \(w_{s}\) weight for surface water index

Here groundwater data is daily data and surface water data reflects grouped flooding events. Because of this, the surface water index is NaN for datetime with no observations. To account for this, we fill these NAs with the weighted GW index as SW Index is 0 in those circumstances

Parameters:
  • gw_index – groundwater index
  • sw_index – surface water index
  • gw_weight – groundwater index weight
  • sw_weight – surface water index weight
Returns:

water suitability index
Return type:

Pandas Series
